유스케이스  ( draw.io에서 다운로드?)

시스템의 하나 이상의 액터 또는 이해관계자에게 관측 가능한 결과를 산출하는 시스템에 의해 수행되는 활동 ( 행위와 관련된 액터간 수행되는 활동을 기술하는 것 )

1. 구성요소
 1) 시스템 - 개발하고자 하는 것 ( 만들고자 하는 프로그램 ) - 네모
 2) 액터 - 클라이언트 , 시스템에 정보를 제공하는 또 다른 시스템 - 졸라맨
 3) 유스케이스 - 사용자 입장에서 본 시스템의 기능(액터에게 제공해야 하는 기능) - 동그라미
 4) 관계 - 액터와 유스케이스 사이의 의미있는 관계 
	4-1 : 연관 - 상호작용이 있음 ( 유스케이스와 액터를 실선으로 연결 )
	4-2 : 의존 - (1) 포함 : 포함되는 유스케이스는 포함하는 유스케이스를 실행하기 위해 반드시 실행해야함
				     ( <<include>> ) 라고 표기
			(2) 의존 : 확장 기능 유스케이스와 확장 대상 유스케이스 사이에 형성되는 관계
				     확장 대상 유스케이스 수행  시 특정 조건에 따라 확장기능 수행하는 경우
				    ( <<extend>> ) 라고 표기한다.
			** 선행되어야 하는 쪽으로 화살표 머리를 두면 된다.
	4-3 : 일반화 : 유사한 유스케이스 또는 액터를 모아 추상화한 유스케이스 & 액터와 연결시켜 그룹을 만들어 			   이해도를 높이기 위한 관계. 추상적인 유스케이스 방향으로 끝부분이 삼각형으로 표현된 화살표
